Error Analysis: Newton Method

The interpolation error for Newton's method is bounded by: 

|f(x) - P(x)| ≤ (1/(n+1)!) * |f^(n+1)(ξ)| * |Π(x - x_i)|

where ξ is some point in the interpolation interval.

Key observations from our tests:

  • Equidistant nodes: Error oscillates severely near interval endpoints due to Runge phenomenon
  • Chebyshev nodes: Error is more evenly distributed and significantly smaller near boundaries
  • Error reduction: As degree increases from 5 to 11, maximum error decreases from ~0.05 to ~0.0002
  • Machine precision: With 11 nodes, errors approach machine epsilon (1.78e-14), indicating convergence limits

The error plots (right subplot in each figure) clearly show that Chebyshev nodes dramatically reduce oscillations compared to equidistant spacing, especially at the boundaries of the interval [-5, 5].

Conclusion

Newton method is superior to Lagrange for iterative polynomial construction and evaluation due to its O(n) complexity versus Lagrange's O(n²). The divided differences approach allows efficient computation of coefficients that maintain numerical stability.

Node selection matters: Chebyshev nodes consistently outperform equidistant spacing in error reduction, particularly evident in the Runge phenomenon where equidistant nodes produce oscillations at interval boundaries.

2. Complexity Analysis of Newton and Lagrange Interpolation

This section analyzes the relationship between grid resolution and the number of arithmetic operations in the Newton and Lagrange interpolation polynomial implementation.

2.1 Newton interpolation method

2.1.1 Grid Resolution Definition

x = np.arange(self.lower, self.upper, 0.05)
Step = 0.05

This creates:

  • Range: -5 to 5 (span = 10)
  • Grid step size: 0.05
  • Number of evaluation points: 10 ÷ 0.05 = 200 points

2.1.2 Operation Count per Evaluation Point

For each grid point, the eval_newton() function evaluates the polynomial.

For 6 interpolation nodes (degree 5 polynomial):

  • Loop iterations: 5
  • Operations per iteration:
    • 2 multiplications (product update)
    • 1 subtraction (x - points[i-1])
    • 1 addition (px += coeffs[i] * product)
Total Operations = k × n

where k is algorithm complexity and n number of iterations

Total per evaluation point:

5 × 4 = 20 operations

2.2 Lagrange Method Complexity Analysis

The Lagrange interpolation method constructs the polynomial directly using basis functions: 

L(x) = Σ(y_i * l_i(x))
where l_i(x) = Π((x - x_j)/(x_i - x_j)) for j ≠ i

Complexity per evaluation point with n nodes:

  • For each basis function l_i(x): (n-1) multiplications and (n-1) divisions
  • For all n basis functions: n × (n-1) multiplications + n × (n-1) divisions
  • Final sum: (n-1) additions

Total Lagrange per point: O(n²) operations

For comparison with Newton (O(n) per point), Lagrange is more computationally expensive as the degree increases. However, when interpolation points are fixed and only evaluation points change, Lagrange can be optimized through precomputation.

2.2.1 Complexity Comparison: Newton vs Lagrange

Method Ops per Point (n=5) Ops per Point (n=11) Scalability
Newton 20 ops 44 ops O(n) - Linear
Lagrange ~60 ops ~264 ops O(n²) - Quadratic

Newton's method is significantly more efficient for high-degree polynomials and large evaluation grids. As polynomial degree increases, Newton interpolation maintains better computational efficiency compared to direct Lagrange evaluation.
